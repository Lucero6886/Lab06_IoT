3. Bản chất Học thuật của HTTP Server và Mạng Cục Bộ

Kiến trúc Client-Server tại Lớp ứng dụng (Layer 7 - OSI)

Trong Lab này, ESP8266 được mô phỏng hoạt động giống hệt như các máy chủ của Google hay Facebook, vận hành dựa trên giao thức phi trạng thái (Stateless Protocol) HTTP:

  1. Mạng Cục bộ (Local Area Network - LAN): Khi NodeMCU kết nối vào Router WiFi, Router (thông qua giao thức DHCP) sẽ cấp cho nó một địa chỉ IP nội bộ (Ví dụ: 192.168.1.15). Không ai ngoài Internet có thể truy cập IP này. Chỉ các thiết bị chung một mạng WiFi mới giao tiếp được với nhau.
  2. Chu trình HTTP Request (Yêu cầu) & Response (Phản hồi):
    - HTTP GET Request: Khi bạn mở Chrome điện thoại, gõ http://192.168.1.15/led_on, Chrome đóng vai trò Client, mở một Socket TCP đến cổng 80 của ESP8266 và gửi gói tin GET.
    - URL Routing (Định tuyến): Vi điều khiển bắt được gói tin, bóc tách URL (Route) là /led_on. Nó tra từ điển và chuyển hướng tới hàm C++ tương ứng để bật chân D6.
    - HTTP Response: Xử lý xong phần cứng, ESP8266 phải đóng gói một thông điệp chuẩn (Bao gồm Header và Body) chứa mã HTML: <html>...</html> rồi trả về. Trình duyệt nhận được sẽ vẽ (Render) lại giao diện. Mọi kết nối sau đó bị ngắt (Stateless).

5. Kiến thức nền (HTTP Codes & mDNS)

  • HTTP Status Codes (Mã Trạng Thái): Là ngôn ngữ giao tiếp giữa Server và Trình duyệt.
    - 200 OK: Báo hiệu Request thành công, kèm theo dữ liệu HTML (Body).
    - 303 See Other: Báo hiệu Server đã xử lý lệnh xong, ra lệnh cho Trình duyệt hãy tải lại trang chủ (Chuyển hướng - Redirect). Rất hữu ích để tránh lỗi người dùng F5 nạp lại lệnh bật đèn 2 lần.
    - 404 Not Found: Khi Client gõ sai đường dẫn URL không có trong Routing.
  • MIME Types (Định dạng nội dung): Trong hàm server.send(200, "text/html", html), chuỗi "text/html" là MIME type. Nó nói cho Chrome biết: "Gói hàng tôi gửi là mã Web, hãy vẽ nó ra". Nếu để là "text/plain", Chrome sẽ chỉ in ra toàn bộ text mã code xấu xí.
  • mDNS (Multicast DNS): IP nội bộ thường hay bị đổi mỗi lần khởi động lại router. Công nghệ mDNS (như Bonjour của Apple) cho phép thiết bị "phát thanh" tên của nó trong mạng LAN. Nhờ vậy, ta có thể gõ http://tram-iot.local thay vì phải nhớ 192.168.1.15.

12. CẤM COPY MÙ QUÁNG: Cơn ác mộng "Nháy Kép" HTML & Code Cổ đại

Sự Lạc Hậu của Mã nguồn Mẫu trôi nổi (Đã được vá lỗi)

Nếu bạn tham khảo các mã nguồn cổ trên mạng hoặc tài liệu cũ, tác giả thường sử dụng lớp thư viện thô sơ WiFiServer client và cố gắng in giao diện HTML bằng hàng chục dòng lệnh nối chuỗi client.println("

...");. Phương pháp đó chứa 2 đòn chí mạng:

1. Kỹ thuật Parse Routing Cổ đại: Tác giả thường dùng hàm request.indexOf("GET /on") để tìm chữ cắt chuỗi thủ công. Cách làm này vô cùng chắp vá, dễ sinh lỗi khi URL bị nối thêm tham số và cấu trúc code rối như tơ vò.

2. Cơn ác mộng Ký tự thoát (Escape Character Hell): Để in được dấu nháy kép " của thuộc tính class trong HTML, sinh viên phải chèn ký tự thoát \" trong C++. Chỉ cần gõ thiếu 1 dấu \, trình biên dịch sẽ ném ra cả ngàn dòng lỗi Syntax Error.

GIẢI PHÁP CHUẨN CÔNG NGHIỆP: Trong bộ mã Phần 13 dưới đây, kiến trúc đã được nâng cấp lên ESP8266WebServer với các API server.on() định tuyến cực kỳ mạnh mẽ. Đặc biệt, giao diện HTML được gói bằng Raw String Literal R"=====(...)=====". Kỹ thuật này giúp trình biên dịch C++ bỏ qua mọi dấu nháy kép bên trong, cho phép Kỹ sư copy nguyên xi 1 file HTML/CSS xịn xò dán thẳng vào vi điều khiển!

13. Phân tích Code Chuẩn: ESP8266WebServer, Routing API & Raw String

Bộ mã nguồn Web Server Nhúng tiêu chuẩn. Chú ý cách tách biệt hoàn toàn Tầng Giao diện (HTML) và Tầng Điều khiển (C++ Handlers).

#include <ESP8266WiFi.h>
#include <ESP8266WebServer.h> // Thư viện Web Server cấp cao
#include <ESP8266mDNS.h>      // Phân giải tên miền LAN
#include "DHT.h"

/* ======================================================================
   1. CẤU HÌNH PHẦN CỨNG & MẠNG
   ====================================================================== */
const char* ssid = "WIFI_CUA_BAN";
const char* password = "MAT_KHAU_WIFI";
const char* mdnsName = "tram-iot";     // Tên miền (Truy cập bằng: http://tram-iot.local)

#define LED_PIN 12       // GPIO12 (D6)
#define DHTPIN 14        // GPIO14 (D5)
#define DHTTYPE DHT11
DHT dht(DHTPIN, DHTTYPE);

// Khởi tạo Server lắng nghe tại Port 80 (Cổng HTTP tiêu chuẩn quốc tế)
ESP8266WebServer server(80);

String ledStateStr = "TẮT"; // Biến Text lưu trạng thái để chèn lên Web

/* ======================================================================
   2. KHỐI GIAO DIỆN FRONT-END HTML/CSS (Raw String Literal)
   ====================================================================== */
// Từ khóa PROGMEM ép mảng dữ liệu HTML này lưu vào bộ nhớ Flash (ROM) 
// thay vì ngốn RAM, giúp vi điều khiển không bị sập (Crash).
const char MAIN_page[] PROGMEM = R"=====(



  
  
  Trạm Giám Sát ESP8266
  


  
BẢNG ĐIỀU KHIỂN LOCAL SERVER

  
  
  

    
⚡ Đèn Chiếu Sáng (D6)

    
Trạng thái thực tế: %LED_STATE%

    
    BẬT ĐÈN
    TẮT ĐÈN
  


  
  

    
🌤 Dữ Liệu Môi Trường

    
Nhiệt độ: %TEMP% °C

    

    
Độ ẩm: %HUM% %

    
(Nhấn F5 hoặc Vuốt xuống để cập nhật Data)

  



)=====";

/* ======================================================================
   3. HÀM XỬ LÝ ĐỊNH TUYẾN (ROUTING HANDLERS)
   ====================================================================== */

// 1. Hàm tạo Trang chủ: Render View và điền dữ liệu động
void handleRoot() {
  float t = dht.readTemperature();
  float h = dht.readHumidity();

  // Chuyển kiểu Float -> String. Bẫy lỗi phần cứng nếu cảm biến chết.
  String tempStr = isnan(t) ? "Lỗi" : String(t, 1);
  String humStr = isnan(h) ? "Lỗi" : String(h, 1);

  // Sao chép chuỗi Giao diện gốc
  String html = MAIN_page;
  
  // VIEW ENGINE SƠ KHAI: Thay thế biến đánh dấu (%) bằng thông số thực
  html.replace("%TEMP%", tempStr);
  html.replace("%HUM%", humStr);
  html.replace("%LED_STATE%", ledStateStr);

  // PHẢN HỒI HTTP 200 OK: Trả về kèm Header MIME Type text/html cho Trình duyệt
  server.send(200, "text/html", html);
}

// 2. Endpoint: Kích hoạt Lệnh Bật Đèn
void handleLedOn() {
  digitalWrite(LED_PIN, HIGH); // Điều khiển Vật lý
  ledStateStr = "ĐANG SÁNG";
  
  // PHẢN HỒI HTTP 303: Lệnh Chuyển Hướng (Redirect)
  // Ép trình duyệt tự động nhảy ngược lại về trang chủ ("/") để tránh lỗi kẹt URL
  server.sendHeader("Location", "/"); 
  server.send(303);
}

// 3. Endpoint: Kích hoạt Lệnh Tắt Đèn
void handleLedOff() {
  digitalWrite(LED_PIN, LOW); // Điều khiển Vật lý
  ledStateStr = "TẮT";
  
  server.sendHeader("Location", "/"); 
  server.send(303);
}

// 4. Bẫy Lỗi HTTP 404 (Trang không tồn tại)
void handleNotFound() {
  server.send(404, "text/plain", "Loi 404: Khong tim thay tai nguyen URL nay tren May chu ESP8266!");
}

/* ======================================================================
   4. SETUP & LOOP
   ====================================================================== */
void setup() {
  Serial.begin(115200);
  dht.begin();
  
  pinMode(LED_PIN, OUTPUT);
  digitalWrite(LED_PIN, LOW);

  // KHỞI TẠO HẠ TẦNG MẠNG
  WiFi.mode(WIFI_STA);
  WiFi.begin(ssid, password);
  Serial.print("\n>> Đang kết nối WiFi");
  while (WiFi.status() != WL_CONNECTED) {
    delay(500);
    Serial.print(".");
  }
  Serial.println("\n>> WiFi Connected.");
  Serial.print(">> ĐỊA CHỈ IP TĨNH CỦA BẠN LÀ: ");
  Serial.println(WiFi.localIP());

  // KÍCH HOẠT TÊN MIỀN MDNS (Trải nghiệm Web mượt mà không cần nhớ IP)
  if (MDNS.begin(mdnsName)) {
    Serial.printf(">> Tên miền mDNS sẵn sàng: http://%s.local\n", mdnsName);
  }

  // ĐỊNH TUYẾN URL (ROUTING API)
  // Đăng ký quy tắc: Nếu Request có Route "/xyz" -> gọi hàm tương ứng
  server.on("/", handleRoot);          
  server.on("/led_on", handleLedOn);   
  server.on("/led_off", handleLedOff); 
  server.onNotFound(handleNotFound);   

  // Bật công tắc nguồn của Server
  server.begin();
  Serial.println(">> HTTP Server Listening on Port 80...");
}

void loop() {
  MDNS.update(); // Duy trì sóng phát thanh tên miền mDNS
  
  // LỆNH TỐI QUAN TRỌNG: Lắng nghe và Vớt các gói tin Request từ Trình duyệt liên tục
  server.handleClient();
}
